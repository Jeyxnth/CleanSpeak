1. WHAT IS A DSP FILTER?
================================================================================

A DSP (Digital Signal Processing) filter removes unwanted frequencies or noise 
from audio signals. It processes audio data mathematically to enhance clarity.

Common DSP Filter Types:
- Noise Gate: Removes silence and background hum
- Spectral Subtraction: Estimates and subtracts noise spectrum from signal
- Wiener Filter: Optimal noise reduction using signal prediction
- Band-Pass Filter: Keeps speech frequencies (300-3400 Hz), cuts the rest
- High-Pass Filter: Removes low-frequency rumble and hum
- Butterworth Filter: Smooth frequency response for audio enhancement

8.1 AUDIO NORMALIZATION (Quality Boost - 1 hour)
----------
PROBLEM: Audio levels vary significantly between uploads
- Quiet audio = Whisper struggles to recognize speech
- Loud audio = Clipping and distortion
SOLUTION: Normalize to perceptual loudness (LUFS - Loudness Units relative to Full Scale)

IMPLEMENTATION:
    import pyloudnorm
    
    # Peak normalization - prevent clipping
    if np.max(np.abs(audio)) > 0:
        audio = audio / np.max(np.abs(audio))
    
    # LUFS normalization - perceptual loudness
    meter = pyloudnorm.Meter(sr)
    loudness = meter.integrated_loudness(audio)
    audio = pyloudnorm.normalize(audio, loudness, -16.0)  # -16 LUFS target

TARGETS:
- -14 LUFS: YouTube standard
- -16 LUFS: Streaming standard (Spotify, Apple Music)
- -20 LUFS to 0 dB: Optimal for speech recognition

DEPENDENCIES TO ADD: pyloudnorm


8.2 SPECTRAL CLEANING - Remove Power Line Hum (1.5 hours)
----------
PROBLEM: 50Hz/60Hz hum from electrical infrastructure degrades transcription
SOLUTION: Remove via notch filter

IMPLEMENTATION:
    from scipy.signal import butter, sosfilt, sosfreqz
    
    def remove_powerline_hum(audio, sr, freq=60.0, Q=30):
        """Remove 50/60 Hz powerline hum using notch filter"""
        nyquist = sr / 2
        w0 = freq / nyquist
        
        # Design notch filter
        sos = butter(4, [w0 - 0.01, w0 + 0.01], btype='bandstop', output='sos')
        audio_filtered = sosfilt(sos, audio)
        return audio_filtered
    
    # Remove DC offset
    audio = audio - np.mean(audio)
    
    # Apply hum removal
    audio = remove_powerline_hum(audio, sr)

BENEFITS: 5-10% improvement in low-noise environments

8.4 ADAPTIVE SPECTRAL GATING (Advanced - 1.5 hours)
----------
PROBLEM: Simple noise reduction removes speech elements
SOLUTION: Adaptive gating that preserves speech frequencies

IMPLEMENTATION:
    def adaptive_spectral_gating(audio, sr, threshold=0.02):
        """Gate audio based on spectral content"""
        D = librosa.stft(audio)
        magnitude = np.abs(D)
        phase = np.angle(D)
        
        # Compute spectral centroid for each frame
        spectral_centroids = librosa.feature.spectral_centroid(S=magnitude, sr=sr)[0]
        
        # Speech typically concentrates in 100-8000 Hz
        mask = magnitude > (threshold * np.max(magnitude, axis=0))
        D_gated = D * mask
        
        audio_gated = librosa.istft(D_gated * np.exp(1j * phase))
        return audio_gated

IMPROVED:
    def generate_context_prompt(audio_path, sr):
        """Generate prompt based on audio characteristics"""
        audio, sr = librosa.load(audio_path, sr=sr)
        duration = len(audio) / sr
        
        # Analyze characteristics
        spectral_centroid = librosa.feature.spectral_centroid(y=audio, sr=sr)
        rms = librosa.feature.rms(y=audio)
        snr = np.mean(rms) / np.std(rms)  # Rough SNR estimate
        
        prompts = []
        
        if duration > 300:  # > 5 minutes = likely formal
            prompts.append("This is a professional recording, lecture, or interview.")
        
        if np.mean(spectral_centroid) < 1500:  # Low frequencies dominant
            prompts.append("Speaker may have low voice, slow speech.")
        
        if snr < 2:  # High noise
            prompts.append("Audio has background noise.")
        
        return " ".join(prompts) or None
    
    prompt = generate_context_prompt(audio_path, sr)
    result = model.transcribe(audio_path, initial_prompt=prompt)

DOMAIN EXAMPLES:
- "This is a podcast about technology."
- "This is a medical consultation with technical terminology."
- "This is a conversation with non-native speakers."

9.5 VAD-BASED CHUNKING FOR LONG AUDIO (2 hours)
----------
PROBLEM: Long recordings (> 5 min) have lower accuracy, hallucination issues
SOLUTION: Split by voice activity, process chunks separately

CURRENT STATE: Infrastructure exists but unused

IMPROVED IMPLEMENTATION:
    # Option 1: Use pyannote-audio (professional VAD)
    # Option 2: Use librosa energy-based VAD
    
    def chunk_by_vad(audio, sr, max_chunk_duration=30):
        """Split audio into voice segments"""
        # Compute energy
        S = librosa.feature.melspectrogram(y=audio, sr=sr)
        energy = np.sqrt(np.mean(S ** 2, axis=0))
        
        # Normalize
        threshold = np.mean(energy) + 0.5 * np.std(energy)
        
        # Find voice frames
        voice_frames = energy > threshold
        voice_samples = librosa.frames_to_samples(np.where(voice_frames)[0], sr=sr)
        
        # Split into chunks
        chunks = []
        chunk_start = 0
        for i, sample in enumerate(voice_samples[:-1]):
            if voice_samples[i + 1] - voice_samples[i] > sr * max_chunk_duration:
                chunks.append(audio[voice_samples[chunk_start]:voice_samples[i]])
                chunk_start = i + 1
        
        if chunk_start < len(voice_samples):
            chunks.append(audio[voice_samples[chunk_start]:])
        
        return chunks
    
    chunks = chunk_by_vad(audio, sr)
    transcripts = [transcribe(chunk) for chunk in chunks]
    full_transcript = " ".join(transcripts)

DEPENDENCIES TO ADD (optional): pyannote-audio>=3.0

    def extract_word_timestamps(result):
        """Extract word timing from Whisper result"""
        words_with_times = []
        
        if "segments" in result:
            for segment in result["segments"]:
                if "words" in segment:
                    # Whisper 1.0+ provides per-word timing
                    words_with_times.extend(segment["words"])
                else:
                    # Fallback: distribute segment time across words
                    words = segment["text"].split()
                    start = segment["start"]
                    end = segment["end"]
                    duration_per_word = (end - start) / len(words) if words else 0
                    
                    for i, word in enumerate(words):
                        words_with_times.append({
                            "word": word,
                            "start": start + i * duration_per_word,
                            "end": start + (i + 1) * duration_per_word,
                            "confidence": 0.85  # Estimated
                        })
        
        return words_with_times

10.1 AUDIO CACHING & DEDUPLICATION (1 hour)
----------
PROBLEM: User uploads identical audio multiple times; reprocessing wastes time
SOLUTION: Hash audio, check cache before transcribing

IMPLEMENTATION:
    import hashlib
    
    def get_audio_hash(audio_path):
        """Generate SHA256 hash of audio file"""
        sha256_hash = hashlib.sha256()
        with open(audio_path, "rb") as f:
            for byte_block in iter(lambda: f.read(4096), b""):
                sha256_hash.update(byte_block)
        return sha256_hash.hexdigest()
    
    audio_hash = get_audio_hash(upload_path)
    
    # Check Supabase

    if existing.data:
